# Handoff — Shared Memory Layer Brainstorm (2026-04-23)

**For:** tomorrow-Trey, tomorrow-Claude, or Codex picking this up.

**Status:** single long brainstorming session produced a coherent end-to-end design. Not yet implemented. Spec captured at `shared-memory-layer-spec-v0.1.md` alongside this file.

## What this is

A harness-agnostic, local-first, daemon-backed shared memory layer that works across every agent harness on the machine (Claude Code, Codex CLI, Cursor, and any other MCP-speaking client) without forking, modifying, or wrapping any of them. Trey's idea, motivated by being one of the world's leading experts on agent memory and realizing that no shipping system connects the dots across harnesses.

The handbook (`agent-harness-memory-context-handbook-v2.2.md`) is the worldview. This spec is the shared-memory instantiation of that worldview with four additions the handbook doesn't have:

1. **Daemon architecture owning all state** — only possible because it's local-first and single-user. Enables live cross-session coordination, which the handbook can't address because OpenClaw is single-device.
2. **Three-layer dreaming pipeline** (substrate / journal / cleanup) — the handbook flagged substrate-vs-journal in one paragraph; this design commits to it as a full architecture, with Pass-3 uncomfortable questions getting their own surface that can be *heard* without promoting to belief.
3. **Weekly Relationship Reality Check** — a user-facing ritual for actively fighting relationship-fact drift, which the handbook named as an unsolved problem with no known production implementation.
4. **OpenAI Privacy Filter integration** (released 2026-04-22, one day before this session) for per-span privacy policy and masked synthesis views.

## What's locked in (don't re-litigate)

These are the 14 design decisions Trey and Claude agreed to in the session. Summarized so you can jump straight to building without re-opening.

1. **Three namespaces, cleanly scoped:** `me`, `project`, `agent`. Global agent pool (one shared cross-harness cross-project pool).
2. **Integration: MCP floor + per-harness enhancement layers.** Universal MCP; richer in Claude Code (hooks), Codex (AGENTS.md + hooks where supported), Cursor (rules); degraded pull-only for the rest.
3. **Storage: Markdown + frontmatter canonical, SQLite as derived index, JSONL event log per device.** Files are source of truth. Index is disposable. Event log is durable audit.
4. **Single local daemon (`memoryd`).** Owns file watcher, indexer, embedder, dreaming scheduler, sync manager, Privacy Filter, event log. MCP clients are thin forwarders over Unix socket.
5. **Identity: implicit git-remote with `.memory-project.yaml` override.** Git remote URL (normalized) → SHA256 → canonical project id. Yaml override for monorepos/workspaces/aliases. Global agent pool for cross-cutting.
6. **Sync: git-backed with semantic frontmatter merge driver.** Custom driver does field-level merge on frontmatter (scalars: newest `updated_at`; arrays: union by id; enum transitions: poset dominance; irreconcilable → quarantine). Body gets textual 3-way merge, conflicts go to quarantine.
7. **Passive recall: per-harness capability mapping (choice E from Q7).** Claude Code: SessionStart base block + UserPromptSubmit delta. Codex: AGENTS.md convention + any startup hooks. Cursor: rules + on-demand. Degraded: pull-only. Base block is byte-identical across turns for cache stability; deltas go in suffix.
8. **Governance: A′ — full handbook machinery built, namespace-scoped YAML policies tune strictness.** Machinery always running: grounding verification, contradiction detection with LLM tiebreak, supersession chains, tombstone matching, sensitivity classification, temporal validity inference, quarantine state. Policies (me-strict, project-standard, agent-strict, dreaming-strict) tune gates per namespace.
9. **Three-layer dreaming: substrate + journal + cleanup.** Substrate is per-device, always-on, 14-day fragment lifetime. Journal runs daily per scope on a leased device (git-synced advisory lease), three passes (why / what should change / uncomfortable question). Only grounded snippets promote; dream prose never promotes. Uncomfortable questions surface in passive recall without promoting. Cleanup is idempotent, any device, safe to run concurrently.
10. **Secrets: four-layer defense + opt-in OpenAI Privacy Filter.** Layer 1: regex+entropy. Layer 2: Privacy Filter (1.5B, Apache 2.0, local inference, eight span labels). Layer 3: storage routing by sensitivity — `secret` refused, `confidential/personal` → age-encrypted tier, `public/internal` → plain git-synced. Layer 4: daemon commit hook re-runs Layer 1 on deltas. Masked synthesis views for dreaming prevent sensitive content leaking into dream prose.
11. **MCP tool surface: 7 agent-facing tools, admin in CLI/slash-commands only.**
    - `memory_search`, `memory_get`, `memory_write`, `memory_supersede`, `memory_forget`, `memory_startup`, `memory_note`.
    - Token-efficient responses, bounded snippets, structured JSON envelope, "want more? call `memory_get`."
    - Admin surface (review, diff, audit, lint, conflicts, rollback, pin, etc.) stays in CLI + slash commands so agents can't accidentally reach for it.
12. **Live cross-session coordination: event-log-driven, three levels.** Level 1 writes-only, Level 2 (default) candidates+notes visible to peers, Level 3 (opt-in) presence + claim locks. Shared substrate pool per device-user-project — all harnesses write to the same substrate file so cross-harness pattern recognition works at synthesis time.
13. **Observability: CLI + TUI (`memory ui`) + local web dashboard + Weekly Relationship Reality Check.** Split by use case: CLI for scripts, TUI for interactive work, web for graphs and rich views. Reality Check runs weekly, surfaces drift-risk-scored relationship facts for confirm/correct/forget/skip.
14. **Internal namespace taxonomy encodes handbook disciplines.** Me-memory: identity/ (role, principles) + relationship/ (facts, preferences, corrections, patterns) + knowledge/ + episodic/ + prospective/. Project-memory: state + decisions + open-questions + playbooks + entities + episodic + invariants + regressions. Agent-memory: patterns + playbooks + postmortems + anti-patterns + heuristics + regressions + episodic. Shared infra: dreams/ + substrate/ + encrypted/ + tombstones/ + events/ + policies/ + leases/.

**Added late in the session:** `regressions/` subtree in both project and agent namespaces. Fingerprinted records of things that broke, with detection signatures. When the agent is about to execute something matching a signature, passive recall fires a "heads up, this pattern burned us on X — see regr_... for fix."

**Late design constraint:** every MCP tool response should be as agent-friendly and token-efficient as humanly possible. Structured JSON, bounded snippets, explicit metadata, "want more? call `memory_get`" as default. This applies to spec-level design of every response shape.

## What's open (for next session)
