## v0.2 erratum: executable canonical-isolation reader contract

Spec v0.2 §12 says:

- `Substrate::read_memory_envelope` on Stream F noncanonical paths returns `NotACanonicalMemory`.

That wording is not executable as written if `Substrate::read_memory_envelope` accepts `&MemoryId`; a `MemoryId` cannot address `dreams/...`, `substrate/...`, `encrypted/substrate/...`, or `leases/...` repository paths.

Executable API contract for implementation:

- `Substrate::read_path_envelope(&RepoPath) -> Result<MemoryEnvelope, ReadError>` is the path-addressed reader.
- For Stream F noncanonical paths, `Substrate::read_path_envelope(&RepoPath)` returns `ReadError::NotACanonicalMemory { path }` before canonical frontmatter parsing.
- Implementers must not add a redundant path reader only to satisfy the impossible `read_memory_envelope(&MemoryId)` wording.
- Canonical memory lookup by `MemoryId` remains scoped to canonical memories.

This matches Task 1 and Task 2 in the implementation plan.
