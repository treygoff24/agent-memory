## Residual risks

- `PeerHeartbeatAck` and `PeerStatusResponse` intentionally expose bounded active-peer/session metadata to local daemon clients. They are not MCP-exposed and are admin/local surfaces, but future harnesses should continue treating `salient_entities` as canonical entity IDs rather than free-form labels or secrets.
- `PeerDeliveryAuditEntry` is safe on the production delta path because summaries are sanitized before recording; future non-delta callers of `HandlerState::record_peer_delivery` must preserve the same `safe_plaintext_fragment` boundary before storing entries.
- Claim-lock contention writes an authorized audit event to the substrate event log. Presence, claim-lock registry state, peer-update cooldowns, and peer-delivery audit remain RAM-only, but any future attempt to make claim locks cross-device visible should receive a fresh privacy/security review before adding a lease/tombstone persistence layer.
- This lane ran focused Stream I security/privacy tests and clippy for `memorum-coordination`/`memoryd`; it did not run the full Task 23 workspace/doc gates.
