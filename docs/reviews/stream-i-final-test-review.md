# Stream I Final Test Review

Status: Changes requested

## Findings

- Severity 1, `docs/specs/stream-i-cross-session-v0.1.md:952-954` / `crates/memoryd/tests/coordination_integration.rs:36-191`, issue: the §11.2 daemon integration matrix is not fully covered. The spec requires daemon-level tests for `test_level2_no_insert_below_threshold` and `test_level2_cap_two_entries`, but the current coordination integration file covers Level 1 suppression, one Level 2 happy path, Level 3 presence, delivery audit, attribution, and cooldown only; the only below-threshold and cap assertions I found are unit/render-level (`crates/memorum-coordination/tests/gate_unit.rs:41-83`, `crates/memoryd/tests/coordination_recall_render.rs:102-110`), not the required `crates/memoryd/tests/` end-to-end path. Why it matters: the final gate requires the §11 acceptance matrix to be fully covered, and these missing daemon tests are the regressions most likely to catch broken candidate gathering, threshold handoff, cap enforcement, and `<pending-attention>` accounting after integration. Suggested fix: add focused daemon integration tests that write a no-overlap peer memory and assert no `<peer-update>`, then write four above-threshold peer memories and assert exactly two rendered `<peer-update>` entries plus the capped pending-attention count.
- Severity 2, `docs/specs/stream-i-cross-session-v0.1.md:929` / `docs/plans/2026-05-01-stream-i-cross-session.md:1401` / `crates/memorum-coordination/tests/gate_unit.rs:161-175`, issue: Tier 3 short-circuit is not tested via a scoring spy. The implementation currently short-circuits before scoring (`crates/memorum-coordination/src/gate.rs:34-49`), and the tests assert empty output plus no surfaced-write mutation, but neither `test_tier3_returns_empty_no_scoring` nor `test_relevance_gate_skipped_for_tier3` has a counter/spy proving that per-write scoring was never invoked. Why it matters: the Stream I final test lane explicitly requires a spy for the Tier 3 no-op path; without it, a regression that scores candidates and then discards the result can still pass the current tests while violating the Tier 3 hot-path and privacy/performance contract. Suggested fix: introduce a narrow test seam for the scorer or a test-only scorer wrapper with an `AtomicUsize` counter, call `evaluate` with Tier 3 and high-scoring candidates, and assert the counter remains zero.
